Q1: 본인의 컴퓨터의 IP 주소는?
220.69.209.221 이며,
이더넷상의 IP는 192.158.1.38 이다.

Q2: 상위 계층 프로토콜의 값은?
ICMP

Q3: IP 헤더의 크기는? IP 데이터그램에서 페이로드의 크기는?
Header Length: 20 bytes
파란 부분이 페이로드이며, Total Length - 페이로드 크기 = 36bytes 입니다.

Q4: 이 IP 데이터그램은 단편화 되었는가? 단편화 여부를 어떻게 알 수 있는가?
단편화 여부는 Fragment Offset의 값으로 알 수 있으며, 값이 0이므로 단편화가 되지 않았음을 알 수 있다. 만약 0이 아니면 단편화가 이루어진 것이다.

Q5: 본인의 컴퓨터에서 ICMP Echo Request 메시지를 보낼 때 마다 변경되는 IP 데이터그램의 필드는 무엇인가?
TTL 값이 바뀌는 것을 알 수 있다

Q6: 이 메시지들 중에서 고정된 필드와 변경되는 필드를 기술하고, 그 이유를 설명하라.
고정필드: 출발지 IP, 도착지 IP
변경필드: TTL: 목적지로 가는 경로 확인을 위함

Q7: 식별자 필드(identification field)의 값의 패턴을 설명하라.
ICMP 프로토콜을 통해 request를 요청할 때마다 1씩 증가한다.
